# AGENTS.md — 에이전트 행동 지침

이 파일은 모든 세션에서 자동으로 로드되는 행동 규칙이다.

## 세션 시작 루틴

1. `memory/MEMORY.md` 로드 — 환경 사실, 교훈, 컨벤션 확인
2. `memory/USER.md` 로드 — kevin의 선호도와 스타일 확인
3. `skills/` 인덱스 확인 — 관련 스킬 있으면 먼저 읽는다

## 작업 완료 후 루틴

작업이 끝나면 반드시:

- **새로운 패턴/절차를 발견했는가?** → `skills/`에 새 스킬 파일 생성 또는 기존 스킬 개선
- **유저에 대해 새로 배운 것이 있는가?** → `memory/USER.md` 업데이트
- **환경/도구에 대해 배운 것이 있는가?** → `memory/MEMORY.md` 업데이트
- **프로젝트 맥락이 생겼는가?** → `projects/` 아래 해당 프로젝트 파일에 기록

## 스킬 작성 규칙

스킬 파일 위치: `skills/<카테고리>/<스킬명>.md`

스킬 파일 구조:
```
# 스킬명

**언제 사용**: 이 스킬을 꺼낼 상황
**마지막 개선**: YYYY-MM-DD
**성공 횟수**: N

## 절차

1. 단계 1
2. 단계 2
...

## 주의사항

- 알려진 함정이나 예외

## 결과 예시

간략한 성공 사례
```

## 메모리 작성 규칙

- **MEMORY.md**: 환경 사실, 프로젝트 컨벤션, 도구 quirk, 교훈. 항목당 1-2줄.
- **USER.md**: kevin의 선호도, 커뮤니케이션 스타일, 기술 수준, 싫어하는 것.
- 중복 추가 금지. 업데이트 시 기존 항목을 교체.

## Curator 루틴 (주간)

`curator/CURATOR_LOG.md`에 기록:
- 사용되지 않는 스킬 → 삭제 또는 병합
- 중복 스킬 → 통합
- 오래된 메모리 항목 → 업데이트 또는 제거
- 가장 많이 쓰인 스킬 → 개선 우선순위 설정
